Generally, the best place for scripts, though, is at the end of the body section. This guarantees that CSS styling and image display won't get held up while scripts are loading. When you embed blocks of JavaScript in HTML (as opposed to having a separate JavaScript file), you must enclose the JavaScript code between <script> and </script> tags.

For most purposes, coders prefer to have all or most of their JavaScript code in a separate JavaScript file, then have the browser load this external file at the same time it's loading the HTML file.

What makes a JavaScript file a JavaScript file is its file extension: .js. The front end of the file name is up to you, as long as it's legal. Any of these would be fine.

  • script.js
  • core.js
  • main-code.js
  • main_code.js
  • maincode.js

You can include as many external JavaScript files as you like.

You can include JavaScript files and also embed other JavaScript code in the HTML.
